# 会议自动化 (Meeting Automation)

> **前置条件：** 先阅读 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md) 了解认证和错误处理。

## 核心能力

本 Skill 提供端到端的会议自动化能力，覆盖会前、会中、会后全流程：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   会前准备   │ -> │   会中支持   │ -> │   会后跟进   │
└─────────────┘    └─────────────┘    └─────────────┘
  - 查询空闲时间       - 会议提醒          - 获取妙记
  - 预订会议室         - 发送材料          - 生成纪要
  - 发送邀请           - 记录出席          - 创建待办
  - 日程创建           - 实时转录          - 发送邮件
  - 材料准备           -                  - 归档知识库
```

## 权限要求

| 操作 | 所需 scope |
|------|-----------|
| 查询日历 | `calendar:readonly` |

| 发送消息 | `im:message` |
| 搜索用户 | `contact:readonly` |

## 会前自动化

### 1. 查询参会人空闲时间

**命令**:
```bash
lark-cli calendar +freebusy --users <user_ids> --start <timestamp> --end <timestamp>
```

**智能推荐算法**:
```
1. 获取所有参会人的忙闲状态
2. 找出共同空闲时间段
3. 排除非工作时间（9:00-18:00）
4. 优先推荐上午时段（精力充沛）
5. 考虑时区差异（如有跨时区参会人）
```

**推荐输出格式**:
```
推荐时间段：
1. 明天 10:00-11:00（所有人空闲）⭐ 推荐
2. 明天 14:00-15:00（张三 14:30 有另一个会）
3. 后天 09:30-10:30（所有人空闲）
```

### 2. 预订会议室

**可用会议室查询**:
```bash
lark-cli api GET /open-apis/vc/v1/rooms --params '{"page_size":"10"}'
```

**会议室选择逻辑**:
1. 容量匹配参会人数（+2 缓冲）
2. 设备需求（投影/视频会议）
3. 地理位置（如有外部参会人）
4. 时间冲突检测

### 3. 创建日程并发送邀请

**创建日程命令**:
```bash
lark-cli api POST /open-apis/calendar/v4/calendars/primary/events --data '{
  "summary": "会议主题",
  "description": "会议议程\n\n1. 议题一\n2. 议题二",
  "start_time": "2026-04-15T10:00:00+08:00",
  "end_time": "2026-04-15T11:00:00+08:00",
  "attendees": [
    {"email": "user1@company.com"},
    {"email": "user2@company.com"}
  ],
  "location": "会议室 A",
  "reminders": [
    {"method": "popup", "minutes": 15}
  ]
}'
```

**邀请模板**:
```markdown
主题：{会议主题}

时间：{日期} {时间}
地点：{会议室}
参会人：{名单}

会议议程：
1. {议题一} - {主讲人} ({时长})
2. {议题二} - {主讲人} ({时长})
3. Q&A ({时长})

会前准备：
- 请阅读 {文档链接}
- 准备 {相关材料}

会议链接：{飞书视频会议链接}
```

### 4. 发送会议提醒

**提醒时机**:
- 提前 1 天：首次提醒
- 提前 1 小时：最后确认
- 提前 15 分钟：紧急提醒

**提醒消息模板**（飞书消息）:
```markdown
📅 会议提醒

{会议主题} 即将在 {时间} 开始

📍 地点：{会议室}
👥 参会人：{出席情况}

📋 议程回顾：
{简要议程}

💡 温馨提示：请提前 5 分钟到场
```

## 会中支持

### 1. 发送会议材料

在会议开始前自动在群聊中发送材料：
```
1. 读取预设的文档链接
2. 生成消息卡片
3. 发送到会议群聊
```

### 2. 记录出席情况

通过扫码或定位记录实际出席人员：
```
出席记录：
- 应到：{人数}
- 实到：{人数}
- 请假：{名单}
```

### 3. 实时转录（可选）

开启飞书妙记实时转录：
- 语音转文字
- 实时字幕
- 关键词提取

## 会后自动化

### 1. 获取飞书妙记

**妙记查询命令**:
```bash
# 查询最近的会议记录
lark-cli vc +list --start <timestamp> --end <timestamp>

# 获取妙记详情
lark-cli vc +get --id <vc_id>
```

### 2. 生成会议纪要

**调用 lark-content-factory**:
```
输入：妙记 URL
输出:
- 结构化会议纪要
- 待办事项列表
- 关键决策汇总
```

**纪要文档结构**:
```markdown
# 会议纪要：{主题}

## 基本信息
| 项目 | 内容 |
|------|------|
| 时间 | {日期时间} |
| 地点 | {会议室} |
| 主持 | {姓名} |
| 记录 | {姓名} |
| 出席 | {名单} |
| 缺席 | {名单} |

## 核心决策
:::callout{emoji="✅" background-color="light-green"}
决策内容 1
决策内容 2
:::

## 讨论摘要

### 议题一
**汇报人**: {姓名}
**主要内容**: {摘要}
**讨论要点**:
- 观点 1
- 观点 2
**结论**: {决议}

### 议题二
...

## 待办事项
| # | 任务 | 负责人 | 截止日 | 优先级 |
|---|------|--------|--------|--------|
| 1 | {任务} | {姓名} | {日期} | P0 |
| 2 | {任务} | {姓名} | {日期} | P1 |

## 附件
- {会议材料链接}
- {妙记链接}
- {录制视频链接}
```

### 3. 创建待办事项

**批量创建任务**:
```bash
# 为每个待办创建飞书任务
lark-cli task +create --title "{任务}" --assignee "{user_id}" --due "{date}"
```

**任务追踪表格**:
```markdown
## 任务追踪看板

### P0 - 紧急重要
- [ ] 任务 1（负责人：@张三，截止：4/20）
- [ ] 任务 2（负责人：@李四，截止：4/18）

### P1 - 重要不紧急
- [ ] 任务 3（负责人：@王五，截止：4/30）
```

### 4. 发送会议总结邮件

**邮件生成并发送**:
```
收件人：全体参会人 + 需知悉人员
主题：【会议纪要】{会议主题} - {日期}

正文:
- 核心决策摘要
- 待办事项列表
- 详细纪要链接
```

### 5. 归档到知识库

**知识库归档**:
```
1. 确定归档位置（项目知识库/部门知识库）
2. 移动或复制纪要文档
3. 更新知识索引
4. 设置适当权限
```

## 使用示例

### 示例 1：组织周会

```
用户：帮我安排下周一的产品周会，参会人有张三、李四、王五

AI 执行流程:
1. 查询三人下周的空闲时间
2. 推荐 3 个时间段供选择
3. 用户确认后创建日程
4. 预订会议室
5. 发送邀请邮件
6. 会前 1 天发送提醒
7. 会后获取妙记生成纪要
8. 创建待办任务
```

### 示例 2：处理已完成会议

```
用户：这是刚才的会议妙记，帮我整理一下
https://waytoagi.feishu.cn/minutes/MObcxxxxx

AI 执行流程:
1. 读取妙记内容
2. 识别会议类型（周会/评审会/脑暴会）
3. 生成结构化会议纪要
4. 提取待办事项（识别"张三负责 xxx"等表述）
5. 在飞书任务中创建任务
6. 发送总结消息给参会人
```

### 示例 3：跨部门协调会

```
用户：组织一个跨部门协调会，参会人分布在多个时区

AI 执行流程:
1. 查询各时区参会人的空闲时间
2. 考虑时差，推荐合适时间（如北京时间 15:00）
3. 创建视频会议（无需物理会议室）
4. 发送含视频会议链接的邀请
5. 会后生成双语纪要（如需要）
```

## 特殊场景处理

### 1. 紧急会议

```
快速流程:
1. 仅检查关键参会人空闲
2. 简化邀请内容
3. 电话/消息直接通知
4. 会后补正式纪要
```

### 2. 例会自动化

```
配置模板:
- 固定时间（如每周一 10:00）
- 固定参会人
- 标准议程
- 自动循环创建
```

### 3. 大型会议（>20 人）

```
特殊处理:
1. 提前 2 周开始协调
2. 使用投票确定时间
3. 准备备用会议室
4. 设置签到处
5. 分组讨论记录
```

### 4. 保密会议

```
安全措施:
1. 限制妙记访问权限
2. 敏感内容人工审核后再分发
3. 会议纪要设置访问权限
4. 会后销毁临时材料
```

## 错误处理

| 错误场景 | 处理方案 |
|----------|----------|
| 找不到共同空闲时间 | 建议分批次开会或异步沟通 |
| 会议室已满 | 推荐附近会议室或改为视频会议 |
| 参会人拒绝邀请 | 通知组织者，建议改期或单独沟通 |
| 妙记生成失败 | 使用文字记录替代，人工整理 |
| 任务创建失败（用户不存在） | 记录任务，手动分配 |

## 性能优化

1. **批量查询**: 参会人空闲时间批量查询
2. **缓存策略**: 用户 ID、会议室信息缓存
3. **并发处理**: 多个待办任务并发创建
4. **异步处理**: 大文档生成走异步

## 最佳实践

### 会议前
- ✅ 提前 2-3 天发送邀请
- ✅ 附上会前阅读材料
- ✅ 明确会议目标和预期产出
- ✅ 设置合理的会议时长（默认 30/60 分钟）

### 会议后
- ✅ 24 小时内发出纪要
- ✅ 待办明确责任人和截止日
- ✅ 纪要抄送需知悉人员
- ✅ 重要决策单独同步给相关方

### 跟进机制
- ✅ 下次会议首先回顾待办完成情况
- ✅ 超期任务自动提醒
- ✅ 周会生成周报汇总
