# 文档增强助手 (Document Enhancer)

> **前置条件：** 先阅读 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md) 了解认证和错误处理。

## 核心能力

本 Skill 分析现有飞书文档内容，智能添加视觉增强元素：

```
输入：飞书文档 URL
  ↓
分析引擎
  ├── 内容类型识别（说明/步骤/对比/数据）
  ├── 结构分析（标题层级/段落长度）
  └── 增强点检测（关键信息/冗长段落）
  ↓
增强输出
  ├── Callout 高亮块（关键信息）
  ├── 表格（数据/对比）
  ├── 分栏（并列内容）
  ├── 画板（流程图/架构图）
  └── 分割线/目录（改善节奏）
```

## 权限要求

| 操作 | 所需 scope |
|------|-----------|
| 读取文档 | `docx:readonly` |
| 更新文档 | `docx:write` |
| 搜索用户 | `contact:readonly` (用于@提及) |

## 增强类型

### 1. Callout 高亮块

**检测规则**:
| 内容特征 | 增强建议 | Callout 样式 |
|----------|----------|-------------|
| "注意"/"请注意" | 警告提示 | yellow + ⚠️ |
| "重要"/"关键" | 重点强调 | orange + 🔑 |
| "提示"/"Tips" | 小贴士 | blue + 💡 |
| "警告"/"危险" | 风险警示 | red + 🚫 |
| "成功"/"完成" | 成功状态 | green + ✅ |
| "最佳实践" | 推荐方案 | green + 🌟 |

**示例转换**:

原文:
```
注意：这个配置一旦设置后不能修改，请谨慎操作。
```

增强后:
```markdown
:::callout{emoji="⚠️" background-color="light-yellow"}
**注意**：这个配置一旦设置后不能修改，请谨慎操作。
:::
```

### 2. 表格转换

**适合转表格的场景**:

1. **多项目对比**
```
原文:
方案 A 价格 100 元/月，支持 10 个用户，存储空间 100GB
方案 B 价格 200 元/月，支持 50 个用户，存储空间 500GB
方案 C 价格 500 元/月，支持 200 个用户，存储空间 2TB

增强后:
| 方案 | 价格 | 用户数 | 存储空间 |
|------|------|--------|----------|
| 方案 A | 100 元/月 | 10 | 100GB |
| 方案 B | 200 元/月 | 50 | 500GB |
| 方案 C | 500 元/月 | 200 | 2TB |
```

2. **参数列表**
3. **时间线/里程碑**
4. **职责分工**

### 3. 分栏布局

**适合分栏的场景**:

1. **对比展示**（两栏）
```markdown
<grid cols="2">
<column>

### 方案 A 优势
- 成本低
- 部署快
- 易上手

</column>
<column>

### 方案 B 优势
- 功能强
- 扩展好
- 支持佳

</column>
</grid>
```

2. **多特性并列**（三栏）
3. **步骤分解**（左步骤右说明）

### 4. Mermaid 画板

**检测规则**:
| 内容特征 | 图表类型 |
|----------|----------|
| 流程描述（第一步、第二步） | flowchart |
| 时序关系（A 发送给 B） | sequenceDiagram |
| 层级结构（包含/组成） | mindmap |
| 状态变化（从 X 到 Y） | stateDiagram |
| 时间安排（第 1 周、第 2 周） | gantt |

**示例转换**:

原文:
```
项目流程如下：
1. 需求评审
2. 技术方案设计
3. 开发实现
4. 测试验证
5. 上线发布
```

增强后:
```mermaid
flowchart LR
    A[需求评审] --> B[技术方案设计]
    B --> C[开发实现]
    C --> D[测试验证]
    D --> E[上线发布]
```

### 5. 分割线与目录

**分割线添加规则**:
- 每个大章节后添加
- 内容类型切换时添加
- 长文档每 2-3 屏添加

**目录生成**:
```markdown
## 目录

- [概述](#概述)
- [功能说明](#功能说明)
  - [核心功能](#核心功能)
  - [技术架构](#技术架构)
- [使用指南](#使用指南)
- [常见问题](#常见问题)

---
```

## 增强策略

### 策略一：轻度增强（默认）

适合：技术文档、API 文档、规范文档

**增强内容**:
- Callout（仅关键信息）
- 分割线（章节分隔）
- 目录（长文档）
- 代码块语法高亮

### 策略二：中度增强

适合：方案文档、产品文档、培训材料

**增强内容**:
- 轻度增强全部内容
- Callout（适度使用）
- 表格（数据/对比）
- 简单画板（流程图）

### 策略三：重度增强

适合：宣传文档、汇报材料、学习笔记

**增强内容**:
- 中度增强全部内容
- 分栏布局（视觉丰富）
- 多种画板（架构图/时序图/思维导图）
- emoji 点缀
- 颜色配置

## 智能检测规则

### 1. 关键信息检测

```python
# 伪代码逻辑
def detect_key_info(paragraph):
    keywords = {
        "warning": ["注意", "警告", "危险", "风险", "谨慎"],
        "important": ["重要", "关键", "必须", "务必"],
        "tip": ["提示", "小贴士", "建议", "推荐"],
        "success": ["成功", "完成", "通过", "达标"]
    }
    
    for category, words in keywords.items():
        if any(word in paragraph for word in words):
            return category
    
    return None
```

### 2. 表格转换检测

```python
def should_convert_to_table(content):
    # 检测对比结构
    if contains_comparison(content):
        return "comparison"
    
    # 检测参数列表
    if contains_parameter_list(content):
        return "parameters"
    
    # 检测时间线
    if contains_timeline(content):
        return "timeline"
    
    return None
```

### 3. 画板生成检测

```python
def detect_diagram_type(content):
    if contains_flow_keywords(content):
        return "flowchart"
    
    if contains_sequence_keywords(content):
        return "sequenceDiagram"
    
    if contains_hierarchy_keywords(content):
        return "mindmap"
    
    if contains_time_keywords(content):
        return "gantt"
    
    return None
```

## 错误处理

| 错误场景 | 处理方案 |
|----------|----------|
| 文档过长（>5000 字） | 分批次处理，按章节更新 |
| 画板语法错误 | 捕获错误，修正后重试 |
| 定位失败 | 扩大定位范围，使用标题定位 |
| 增强后格式错乱 | 回滚到原版本，降级增强 |

## 性能优化

1. **增量更新**: 只更新变更部分，保留原有图片和评论
2. **批量写入**: 多个增强元素批量写入（50 块/批）
3. **并行处理**: 独立章节并行处理
4. **缓存**: 文档结构缓存，避免重复读取

## 最佳实践

### 增强原则
- ✅ 内容为王，增强为辅（不过度装饰）
- ✅ 保持一致性（Callout 样式统一）
- ✅ 预留人工审核环节
- ✅ 保留原文档核心内容

### 视觉设计
- ✅ Callout 颜色不超过 3 种
- ✅ 表格行数不超过 15 行（过长分页）
- ✅ 分栏不超过 3 栏（移动端友好）
- ✅ emoji 适度使用（正式文档克制）

### 兼容性考虑
- ✅ 移动端显示效果
- ✅ 打印友好（避免过多颜色）
- ✅ 无障碍阅读（不过度依赖颜色）

## 增强效果评估

### 可读性指标

| 指标 | 计算方式 | 目标值 |

| 信息层次 | 标题层级深度 | 2-4 级 |
| 段落长度 | 平均字数/段落 | <200 字 |
