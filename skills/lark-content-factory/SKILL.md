# 内容工厂 (Content Factory)

> **前置条件：** 先阅读 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md) 了解认证和错误处理。

## 核心能力

本 Skill 支持将单一输入源转换为多种输出格式：

### 输入源类型
1. **飞书妙记 URL** - 会议/采访录音转文字
2. **飞书文档 URL** - 已有文档内容
3. **直接文本** - 用户提供的文字内容

### 输出格式
| 输出类型 | 用途 | 目标位置 |
|----------|------|----------|
| 会议纪要 | 正式会议记录 | 飞书文档 |
| 待办事项 | 任务追踪 | 飞书任务 |
| 新闻稿 | 对外宣传 | 飞书文档/公众号 |
| 社交媒体文案 | 小红书/微博风格 | 飞书文档 |
| 邮件通知 | 内部/外部通知 | 飞书邮箱草稿 |
| 知识库文档 | 沉淀归档 | 飞书知识库 |

## 命令参考

```bash
# 查询可用的 API
lark-cli docs +fetch --doc <doc_url>
lark-cli task +list
lark-cli calendar events instance_view
```

## 权限要求

| 操作 | 所需 scope |
|------|-----------|
| 读取文档 | `docx:readonly` |
| 创建文档 | `docx:write` |
| 创建任务 | `task:write` |
| 读取妙记 | `vc:readonly` |
| 发送邮件 | `mail:write` |

## 工作模式

### 模式一：会议纪要生成

**输入**: 飞书妙记 URL 或会议文字记录

**输出**:
1. 结构化会议纪要文档
2. 待办事项列表（飞书任务）
3. 会议决策汇总

**会议纪要模板**:
```markdown
# 会议纪要：{会议主题}

## 基本信息
- 会议时间：{日期}
- 参会人员：{名单}
- 会议类型：{周会/评审会/脑暴会}

## 核心决策
:::callout{emoji="✅" background-color="light-green"}
关键决策内容
:::

## 讨论要点

### 议题一：{标题}
**背景**: {简述}
**讨论**: {要点}
**结论**: {决策}

### 议题二：{标题}
...

## 待办事项
| 任务 | 负责人 | 截止日期 | 优先级 |
|------|--------|----------|--------|
| {任务描述} | {姓名} | {日期} | P0/P1/P2 |

## 参考资料
- {相关文档链接}
- {提及的文档/@的人}
```

### 模式二：内容多平台分发

**输入**: 原始内容（文章/采访稿/产品发布稿）

**输出**:
1. 小红书风格文案（emoji + 分段 + 标签）
2. 微博风格文案（简短 + 话题）
3. 公众号文章（正式 + 深度）
4. 内部通讯稿

**各平台风格要点**:

**小红书**:
- 标题：15-20 字，含 emoji，制造悬念/痛点
- 正文：短段落，多 emoji，口语化
- 标签：5-10 个相关 hashtag

**微博**:
- 140 字以内
- 带话题 #xxx#
- @相关账号

**公众号**:
- 标题：两段式，信息量大
- 正文：结构化，小标题清晰
- 结尾：行动号召

### 模式三：采访稿整理

**输入**: 采访录音转文字

**输出**:
1. 采访全文整理稿（飞书文档）
2. 金句摘要卡片
3. 社交媒体预告文案

## 使用示例

### 示例 1：处理会议妙记

```
用户：帮我处理这个会议妙记，生成会议纪要和待办
https://waytoagi.feishu.cn/minutes/MObcxxxxxxxxx

AI 执行步骤:
1. 读取妙记内容（lark-cli minutes +fetch）
2. 识别会议类型和关键信息
3. 生成结构化会议纪要文档
4. 提取待办事项创建飞书任务
5. 返回文档链接和任务列表
```

### 示例 2：内容多平台分发

```
用户：把这篇产品发布稿改成小红书和微博文案
{粘贴原文或提供文档链接}

AI 执行步骤:
1. 读取原始内容
2. 分析核心卖点和目标受众
3. 生成小红书版本（emoji + 分段）
4. 生成微博版本（140 字 + 话题）
5. 创建对比文档展示各版本
```

### 示例 3：采访稿处理

```
用户：整理这个采访，输出正式稿和金句卡片
https://example.com/interview-transcript

AI 执行步骤:
1. 分析采访结构（问答段落）
2. 提取关键观点和金句
3. 生成结构化采访稿
4. 创建金句卡片（Callout 样式）
5. 生成社交媒体预告
```

## 内容处理原则

### 1. 保持原意
- 不歪曲原始内容
- 重要信息保留引用
- 数据需核实

### 2. 风格适配
- 正式场合用正式语气
- 社交媒体用轻松语气
- 保持品牌一致性

### 3. 视觉优化
- 合理使用 Callout 高亮
- 关键信息加粗
- 适当使用 emoji（正式文档克制）

### 4. 可追溯
- 保留原始来源链接
- 标注 AI 生成内容
- 重要决策需人工确认

## 错误处理

| 错误场景 | 处理方案 |
|----------|----------|
| 妙记无法读取 | 检查权限，尝试备用链接 |
| 内容过短无法分析 | 请求用户提供更多上下文 |
| 敏感内容 | 标记需人工审核 |
| 任务创建失败（找不到用户） | 使用 @提及代替任务分配 |

## 性能优化

1. **批量创建**: 待办事项批量创建（限 10 个/批）
2. **并发处理**: 多个输出文档并发创建
3. **缓存**: 用户 ID、文档结构缓存

## 最佳实践

### 1. 会议纪要
- 24 小时内发出
- 待办明确责任人和截止日
- 决策部分用 Callout 突出

### 2. 内容分发
- 先出草稿，人工确认再发布
- 各平台发布时间错开
- 追踪各平台反馈

### 3. 采访整理
- 保留受访者原话
- 敏感内容打码或省略
- 发布前需受访者确认

## 扩展能力

### 自定义模板

用户可提供自定义模板：
```markdown
:::template
{用户自定义模板内容}
:::
```

### 多语言支持

支持中英文内容处理：
- 自动检测语言
- 支持翻译输出
- 保持术语一致
