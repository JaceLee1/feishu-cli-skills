## 一、现有资源分析

### 3.2 内容创作类 Skill（P0 - 大赛核心）

#### 3.2.1 `lark-content-factory` - 内容工厂

```markdown
---
name: lark-content-factory
description: 多格式内容生成工厂。将单一输入转换为多种输出格式（会议纪要、新闻稿、社交媒体、邮件等）。当需要批量生成多种内容时使用。
---

核心能力:
1. 输入：会议录音/文字记录/采访稿
2. 输出：
   - 会议纪要（飞书妙记格式）
   - 待办事项（飞书任务）
   - 新闻稿/公众号文章
   - 社交媒体文案（小红书/微博风格）
   - 邮件通知
   - 知识库沉淀文档

使用场景:
- 会议/采访后的内容分发
- 一次输入，多平台发布
- 内容标准化处理
```

#### 3.2.2 `lark-meeting-automation` - 会议自动化

```markdown
---
name: lark-meeting-automation
description: 端到端会议自动化。从安排会议到生成纪要和待办追踪的全流程管理。当需要自动组织会议或处理会后事务时使用。
---

核心能力:
1. 会前:
   - 查询参会人空闲时间 (calendar +freebusy)
   - 创建日程并发送邀请
   - 预订会议室
   - 发送会议提醒

2. 会后:
   - 获取飞书妙记和文字记录
   - 自动生成会议纪要
   - 提取待办事项到飞书任务
   - 发送会议总结邮件
   - 归档到知识库

使用场景:
- 周会/月会自动化
- 跨部门会议组织
- 客户拜访记录
```

#### 3.2.3 `lark-doc-enhancer` - 文档增强助手

```markdown
---
name: lark-doc-enhancer
description: 智能文档优化。为现有飞书文档添加 Callout、表格、分栏、画板等增强元素。当需要美化或增强文档可读性时使用。
---

核心能力:
1. 文档分析：识别文档结构和内容类型
2. 智能增强:
   - 为关键信息添加 Callout 高亮块
   - 将列表转换为表格
   - 添加目录导航
   - 为流程描述生成 Mermaid 画板
   - 插入相关图示

3. 样式优化:
   - 统一标题层级
   - 添加分割线改善节奏
   - 配置颜色主题

使用场景:
- 技术文档美化
- 方案文档专业化
- 学习笔记整理
```

### 3.3 工作流自动化类 Skill（P0 - 大赛核心）

#### 3.3.1 `lark-project-setup` - 项目一键 Setup

```markdown
---
name: lark-project-setup
description: 新项目一键初始化。自动创建知识库、多维表格看板、项目群聊、任务模板等完整项目基础设施。当启动新项目需要快速搭建协作环境时使用。
---

核心能力:
1. 知识库结构:
   - 创建知识空间/文件夹
   - 建立文档目录（项目概览、需求文档、技术方案、会议纪要）
   - 初始化模板文档

2. 多维表格:
   - 项目任务看板（字段：任务名、负责人、状态、优先级、截止日期）
   - 风险登记册
   - 需求池

3. 协作配置:
   - 创建项目群聊
   - 邀请项目成员
   - 设置群公告和群菜单

4. 任务模板:
   - 创建标准任务清单
   - 设置周期性任务

输出:
- 项目知识库 URL
- 多维表格 URL
- 项目群聊 ID
- 项目手册文档
```

#### 3.3.2 `lark-onboarding-flow` - 新员工入职流程

```markdown
---
name: lark-onboarding-flow
description: 新员工入职自动化。创建欢迎文档、发送入职清单、安排培训课程、添加相关群聊。当有新同事入职需要自动化办理入职手续时使用。
---

核心能力:
1. 入职文档:
   - 欢迎文档（含团队介绍、办公指南）
   - 个人任务清单（30-60-90 天计划）

2. 日程安排:
   - 入职培训课程（产品/技术/流程）
   - 1v1 会议（主管/HR/导师）

3. 权限配置:
   - 添加项目群聊
   - 开通知识库访问权限
   - 分配任务

4. 进度追踪:
   - 入职检查清单
   - 试用期目标设定
```

   - 数据汇总文档

## 六、技术实现细节

### 6.1 通用错误处理

```markdown
所有 Skill 统一错误处理策略:

1. API 调用失败:
   - 重试机制（3 次，指数退避）
   - 友好的错误提示
   - 建议修复方案

2. 权限不足:
   - 明确提示所需 scope
   - 提供授权链接

3. 数据验证:
   - 输入参数校验
   - 输出结果验证

4. 日志记录:
   - 每次调用的 log_id
   - 关键决策点记录
```

### 6.2 性能优化策略

```markdown
1. 批量操作:
   - 文档创建：批量写入 blocks（50 块/批）
   - 消息发送：并发发送（限 10 并发）

2. 缓存策略:
   - access_token 缓存（2 小时）
   - 用户 ID 缓存（通讯录）
   - 文档结构缓存

3. 异步处理:
   - 大文档更新走异步任务
   - 长耗时操作返回 task_id
```
